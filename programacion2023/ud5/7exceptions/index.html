    • unchecked: La excepci√≥n no te obligar√° a controlarla con try-catch. Crearemos una excepci√≥n que herede de RuntimeException
    • checked: La excepci√≥n se tiene que controlar con try-catch. Crearemos una excepci√≥n que herede de Exception

    La clase Clientes controla el uso correcto mediante excepciones. Ser√° el usuario de la clase quien controle los errores de cara a mostrar mensajes por pantalla o enviar el error a un fichero log

    public class Clientes {

    private String[] clientes;
    //lleva el total de clientes actual
    private int contador = 0;

    public Clientes(int tama√±o) {
        clientes = new String[tama√±o];
    }
    //indicamos en la firma que hay que tratar la excepciones
    public void agregarCliente(String nombre) 
        throws ClienteYaExisteException, NoHayEspacioException{



        // comprobar espacio
        if (contador >= clientes.length) {
            //lanza excepci√≥n
            throw new NoHayEspacioException("No queda espacio para a√±adir: " + nombre);
        }

        // comprobar si ya existe
        for (int i = 0; i < contador; i++) {
            if (clientes[i].equalsIgnoreCase(nombre)) {
                //lanza excepci√≥n
                throw new ClienteYaExisteException("El cliente '" + nombre + "' ya existe.");
            }
        }
        // agregar cliente
        clientes[contador] = nombre;
        contador++;
    }

    public void mostrarClientes() {
        System.out.println("Lista de clientes:");
        for (int i = 0; i < contador; i++) {
            System.out.println("- " + clientes[i]);
        }
    }
}
