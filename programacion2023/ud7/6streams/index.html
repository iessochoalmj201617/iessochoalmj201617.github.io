    Streams¶

    Los Streams fueron introducidos en Java 8 para abrir la puerta a la programaci√≥n funcional al igual que con las expresiones lambda.

    La API Stream permite manipular las colecciones como nunca antes. Nos permite realizar operaciones sobre la colecci√≥n, como por ejemplo, buscar, filtrar, reordenar, etc.

    Con Streams podemos utilizar cualquier clase que implemente la interfaz Collection como si fuese un Stream con la ventaja que nos ofrecen las expresiones lambda.

    Con streams hay que tener el cuenta que la fuente o colecci√≥n que utilicemos no se puede modificar y no debe afectar al estado de la misma. Cada operaci√≥n dentro del stream debe verse como una operaci√≥n independiente que opera sobre el argumento (colecci√≥n).

    A trav√©s del API Stream podemos trabajar sobre colecciones como si estuvi√©ramos realizando sentencias SQL pero de una manera limpia y clara, evitando bucles y algoritmos que ralentizan los programas e incluso hacen que el c√≥digo se torne inmanejable.

    Cada operaci√≥n del stream debe verse como un paso independiente, es decir, no se puede usar variables intermedias.

    3. Operaciones terminales, como por ejemplo los m√©todos max, min, forEach, findFirst etc.
